Ooyala-Video Cloud Ingestion Adapter: Manifest File Formats

This topic covers the manifest file formats you may use for ingesting videos and assets.
Note: this topic applies only to Ooyala customers migrated to Video Cloud.

See other Ooyala to Brightcove Migration Guides.

Introduction

When ingesting videos, you can use manifest files to specify extra metadata (including the video title, description, flight times, labels, and custom metadata), as well as thumbnail and closed caption files. Manifest files are in either XML or CSV format. For details on manifests, see:

  • CSV Manifest Files
  • XML Manifest Files

When processing begins

Note: This topic applies to OIS v2.5.
Note: Video Cloud does not begin processing any of the videos until the XML or CSV manifest file is received. As a result, it is important to choose a batch upload strategy. If your videos must be available quickly (e.g., news distribution), you should create very small batches or even batch them individually (one CSV or XML file per video).
